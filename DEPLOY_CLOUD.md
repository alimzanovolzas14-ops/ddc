# Запуск в облаке за ~20 минут (онлайн + PostgreSQL)

Цель: чтобы сайт был **онлайн по ссылке**, данные писались в **настоящую базу
PostgreSQL**, и всё работало во время презентации. Готовый `Dockerfile` уже
лежит в проекте — платформа соберёт и запустит его сама.

> Это вариант для демонстрации. Реальное «боевое» размещение портала Нацбанка
> (внутренний/казахстанский защищённый контур) — отдельный шаг после презентации.

Рекомендую **Railway** — не «засыпает» (важно для живого показа) и PostgreSQL
подключается одной кнопкой. Ниже — Railway, а в конце альтернатива (Render).

---

## Вариант A. Railway (рекомендую)

### 1. Загрузите проект на GitHub
Проще всего — через сайт, без командной строки:
1. Зарегистрируйтесь на **github.com**, нажмите **New repository**, назовите
   `ddc-portal`, создайте.
2. На странице репозитория: **Add file → Upload files**. Перетащите папки
   `DDC_site` и `backend`, а также файлы `Dockerfile` и `.dockerignore`
   (они в корне архива). Нажмите **Commit changes**.

### 2. Разверните на Railway
1. Зарегистрируйтесь на **railway.com** (можно через тот же GitHub).
2. **New Project → Deploy from GitHub repo →** выберите `ddc-portal`.
   Railway увидит `Dockerfile` и начнёт сборку.

### 3. Добавьте базу данных (одной кнопкой)
1. В проекте: **New → Database → Add PostgreSQL**.
2. Railway сам создаст переменную `DATABASE_URL` и передаст её вашему сервису —
   вручную ничего копировать не нужно. Сервер увидит её и переключится на
   PostgreSQL (создаст таблицы и демо-данные сам).

### 4. Задайте секрет
В сервисе портала: **Variables → New Variable**
- имя: `DDC_SECRET`
- значение: длинная случайная строка (например, из `openssl rand -hex 32`,
  или любые 40+ случайных символов).

### 5. Получите публичный адрес
В сервисе: **Settings → Networking → Generate Domain**. Появится ссылка вида
`https://ddc-portal-production.up.railway.app`.

### 6. Готово
Откройте эту ссылку — портал онлайн, данные пишутся в PostgreSQL. Проверьте:
войдите в двух окнах (`a.arinova/demo` и `m.amardinov/demo`) и напишите в общий
чат — сообщение появится у второго мгновенно.

**Стоимость:** у Railway стартовый пробный кредит; для нескольких дней демо
этого хватает, дальше — небольшая плата по факту использования.

## Вариант B. Render

1. Зарегистрируйтесь на **render.com**, залейте тот же GitHub-репозиторий.
2. **New → Web Service →** выберите репозиторий. Render определит `Dockerfile`.
3. **New → PostgreSQL** — создайте базу, скопируйте её **Internal Database URL**.
4. В веб-сервисе: **Environment → Add** переменные:
   - `DATABASE_URL` = скопированный адрес базы;
   - `DDC_SECRET` = случайная длинная строка.
5. Deploy. Render даст ссылку `https://...onrender.com`.

⚠️ На **бесплатном** тарифе Render сервис засыпает после 15 минут простоя и
просыпается 30–50 секунд — для живой презентации это неудобно. Варианты:
открыть сайт за пару минут до показа (успеет проснуться), либо взять платный
тариф Starter (~$7/мес, без засыпания). Бесплатная база живёт 90 дней.

## Если что-то не так

- Логи сборки/работы видны в дашборде платформы (вкладка Logs/Deployments).
- Если чат/вход не работают — почти всегда не добавлена база: проверьте, что
  переменная `DATABASE_URL` есть у сервиса.
- Проверка «жив ли сервер»: откройте `https://ВАШ-АДРЕС/api/health` — должно
  вернуть `{"ok":true,...}`.

## Перед показом

- Смените демо-пароли, если данные не совсем «игрушечные».
- Откройте сайт заранее и прокликайте сценарий, который будете показывать.
