# Как запустить портал онлайн (коротко)

Код готов. Чтобы сайт заработал постоянно, нужен **сервер** — включённая
24/7 машина с адресом в сети. Сервер нельзя «сделать кодом», его берут в
аренду (или используют внутренний сервер организации). Это единственный шаг,
который делаете вы. Всё остальное — один скрипт.

## Шаг 1. Возьмите сервер

Любой из вариантов:
- **Бесплатно (для демо):** Oracle Cloud «Always Free» — маленький сервер,
  работающий постоянно (при регистрации спросят карту для проверки, деньги не
  списывают). Выберите образ **Ubuntu 22.04/24.04**.
- **Платно/надёжно:** любой VPS (в т.ч. казахстанский) с Ubuntu — от нескольких
  долларов в месяц.
- **Боевой вариант для Нацбанка:** внутренний сервер организации (согласуйте с
  ИБ — данные сотрудников нельзя хранить на произвольном зарубежном облаке).

Вам нужны: адрес сервера (IP) и вход по SSH.

## Шаг 2. Загрузите проект и запустите установщик

Со своего компьютера скопируйте распакованный архив на сервер и запустите
один скрипт:

```bash
# 1) скопировать проект на сервер (замените IP)
scp -r DDC_site backend  ubuntu@АДРЕС_СЕРВЕРА:/tmp/ddc

# 2) зайти на сервер
ssh ubuntu@АДРЕС_СЕРВЕРА

# 3) запустить установщик
cd /tmp/ddc
sudo bash backend/deploy/install.sh
```

Скрипт сам поставит Node.js, PostgreSQL, создаст базу (с автоматически
сгенерированными паролем и секретом), настроит автозапуск и Nginx. В конце
покажет адрес, по которому открывается портал.

## Шаг 3. (Необязательно) Домен и HTTPS

Если есть доменное имя, направленное на IP сервера, запустите установщик так —
он сам выпустит HTTPS-сертификат:

```bash
sudo DOMAIN=portal.example.kz LETSENCRYPT_EMAIL=you@example.kz bash backend/deploy/install.sh
```

Без домена сайт откроется по `http://АДРЕС_СЕРВЕРА` — этого достаточно, чтобы
показать, что всё работает (включая realtime-чат).

## Что дальше

- Проверьте: откройте адрес в двух окнах под разными пользователями
  (`a.arinova/demo` и `m.amardinov/demo`) и напишите в общий чат — сообщение
  появится у второго мгновенно.
- **Перед реальным использованием** смените демо-пароли и, для боевого запуска,
  используйте HTTPS от доверенного центра сертификации. Полный чек-лист и
  ручная (не автоматическая) инструкция — в `backend/deploy/DEPLOY.md`.

## Что я не могу сделать за вас

Поднять и оплатить сам сервер — для этого нужны ваши аккаунт и доступ. Как
только сервер есть, всё остальное делает установщик из шага 2.
